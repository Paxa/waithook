Web Hook to Websocket Transmiting Service

Waithook was build to ease receiving HTTP notifications in development and testing environments. It uses WebSockets to deliver messages, so it can work behind firewall, network routes and proxy. Think of it as Pub/Sub for HTTP callback.

1. Open WebSocket connection to ws://waithook.com/$your_unique_name
2. Send request to http://waithook.com/$your_unique_name
3. Receive message via WebSocket connection
